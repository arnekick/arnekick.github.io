Stage

Angular developer bij E.care

Tijdens het laaste jaar van mijn studies in de Toegepaste Informatica van Thomas More liep ik gedurende 13 weken stage bij het bedrijf E.care. E.care is een bedrijf dat software ontwikkelt, verkoopt en implementeert die het medisch proces in ziekenhuizen ondersteunt. Hun software concentreet zich vooral op de afdelingen Spoedgevallen en Operatiekwartier. E.care vind het nodig om hun software webbased te maken zodat hun software performanter wordt en er frisser gaat uitzien.

Mijn stageopdracht bestond uit het omzetting van bestaande modules van de E.care software naar hun web based variant in Angular voor zowel computer als tablet. Voor elk van deze bestaande modules moest er een webbased variant worden gerealiseerd. Tijdens de stage hebben we 5 modules in Angular gerealiseerd. Telkens wanneer we aan een nieuwe module begonnen moest er eerst een prototype worden gemaakt, hieruit werden dan de beste aspecten samengevoegd. Zo onstond dan de applicatie die we telkens gingen ontwerpen.

Tijdens mijn stage hebben we gemerkt dat sommige stukken code in meerdere applicaties gebruikt worden. Ik heb dan de opdracht gekregen om uit te zoeken hoe ik een library opbouw in Angular en deze kan publiceren. Hierbij heb ik veel geleerd want dit hadden zowel de developpers van E.care als ikzelf nog nooit gedaan. Nadat ik de library had gemaak t heb ik documentatie moeten schrijven over alle how to's die hierbij kwamen kijken en de errors die ik heb gehad tijdens het opbouwen van de library. Op deze manier weten zij ook hoe ze in de toekomst de library nog kunnen uitbreiden zonder dit allemaal opnieuw uit te zoeken.

Ik ben tijdens mijn stage nog een paar andere frameworks/libraries onderzocht om te kijken of deze voldeden aan de eisen die nodig waren. Zo heb ik de EO.WebBrowser van EssentialObject onderzocht en de Web documenteditor voor Angular van TxTextControl.

Project 4.0

Ik heb samen met een team de opdracht gehad van Ometa om voor duracel het volgend eprobleem op te lossen. Duracell zou graag hun productie willen monitoren voor in het geval er iets mis gaat. De huidige situatie is dat duracell de productie even still legt iedere dinsdag voormiddag voor onderhoud. Hierbij controleren de werknemers dan alles en zoeken ze naar een probleem dat er misschien niet is.
Op het moment dat er iets fout loopt in de machine zouden ze graag een melding krijgen op een app en/of smartwatch van de juiste werknemer, met de info over welke machine defect is en wat er mis is. Hierdoor hoeven ze de producite dinsdag niet meer stil te leggen en verliezen ze geen tijd.
Dust collector: controle gebeurt nu elke dinsdag: alert a filter vol is (laser sensor). Zink kanaal: korrels geraken verstopt in kanaal: alert als kanaal verstopt zit (gewicht voor en na wordt gemeten).

In het onderstaande filmpje vind u een demo van hoe de sensors te werk gaan.

Spectralink Telefoons

Tijdens mijn weekend werk ben ik een project gaan realiseren bij het bedrijf didak injections. Dit project bestond uit het uitrollen van een nieuw communicatiesysteem d.m.v spectralink telefoons en skype for business. Hierbij moest ik telkens de telefoon activeren, deze toevoegen aan het netwerk, een account aanmaken voor de gebruikers, hier een nummer voor de gebruiker aanmaken en aan zijn/haar account koppelen ook moest ik een gemeenschappelijke telefoonboek aanmaken zo is het later eenvoudiger om nummer aan iedereen door te spelen. ja voegt hieraan dus een nummer toe en dan worden deze automatisch doorgepushed naar al de telefoons in het netwerk. Nog een belangrijk punt in het proces was dat ik een contract moest opstellen voor de ontvangst van de nieuwe telefoon in kwestie. Als laatste heb ik ook een handleiding voor het toestel gerealiseerd die duidelijk te begrijpen is voor de minder IT-aangelegde personen binnen het bedrijf.

Indien er problemen waren heb ik contact opgenomen met vanroey deze mensen hebben me telkens verder kunnen helpen met mijn probleem bv. een van de accounts had niet de juiste rechten vanuit de AD meegekregen dus deze kan ik niet toevoegen aan de SfB hierbij heb ik even contact met hun opgenomen.
